운영 환경
-운영 환경은 예상 불가한 사용량에 유연한 대비를위해 Azure virtual machine을 사용한다.
운영체제는 ubuntu 18.04
데이터베이스와 프론트엔드 웹서버는 내부에서 서버와 가상 네트워크 망으로 연결하여 사용자에게 제공돈다.
한국 지역 타겟으로 서비스하기에 빠른 응답을 위해 ASia region으로 지역을 선택한다.

설계 및 구현 제약사항

기능 이외의 다른 요구사항
성능 요구사항
클라이언트에 전송되는 모든 사용자 로드타임은 2.0초를 넘지 않도록 처리합니다
특정 시간대의 과부하를 방지하고자 동시처리가 가능하게 처리한다.

안전 요구사항
ISMS-P 개인정보 인증체계를 지킨다.
//

보안 요구사항
클라이언트와의 통신은 이용 정보에 대해 SHA256을 통해 암호화하여 전송
입력에는 XSS 공격방지와 SQL인젝션 방지를 위한 처리한다.
내부 서버 간의 처리는 가상 네트워크망을 위한 proxy를 이용해 처리하여 외부접속이
방지되도록 처리한다.
사용자의 패스워드 변경 요청 이후 최초 로그인 시 자동 할당된 암호를 즉시 변경해야한다.
//

기능 이외의 다른 요구사항

소프트웨어 품질 특성
모든 클라이언트 서버간 통신은 사용자 로드타임 당 2.0초 이내로 수행되게 한다.

다른 요구사항
동일한 응답 연산의 전송은 캐시 데이터를 이용하여 자원소모를 줄이며 응답시간을 단축시킨단.
